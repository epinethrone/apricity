Settings

Display

Appearance, interface visibility, and motion preferences.

Theme Match the system, or force light/dark. Stays in sync with the top-bar theme button.

Interface

Tools panel Reveal the Tools button in the top bar (power-user MemPalace operations: tunnels, diary, KG admin, maintenance).
Knowledge graph visualisation Adds a Graph tab to the KG panel — a force-directed graph of facts. Off by default; the physics is CPU-heavy on a Pi.
Footer info Memory + fact + tunnel counts and system info (CPU, disk, version). Off gives a cleaner bottom edge — useful while screen-sharing.
Always show panel controls The Memories and Detail panel maximize/close buttons stay opacity 1 at all times. Off (default) hides them until you hover the parent panel — macOS-traffic-light pattern, less visual clutter at rest. Keyboard Tab still reveals them when focus enters either panel.

Memories list

Adaptive browse mode Memories panel scales its browse-grid to the current list — short lists hide the Browse toggle (just scroll), 11–24 memories show 2 columns, 25–50 show 3, 51+ show 4. Off restores the always-on toggle and fixed 4-column grid.

Text display

Polish titles & names Display layer prettifies stored text: drawer titles get Title-Cased and trailing (YYYY-MM-DD) stripped, wing/room slugs render as Home Assistant instead of home-assistant (acronyms like HA, API, TV stay all-caps), author names get re-capitalized. Off shows the raw stored values. Storage is never modified either way.
Relative time Card kickers and the meta-strip Updated cell show short relative phrases (2h ago, yesterday, 3d ago) when recent. Off renders the absolute recorded date (28 May 2026) for every age. The detail panel's Filed cell is always absolute regardless of this setting.

Motion

Reduce motion Disables the search-ring rotation, gear spin, waterfall cascade, and theme cascade. Independent from the system-wide prefers-reduced-motion.

Notifications

Bell behavior, sound, and bulk-clear of the recently-updated marker.

Suppress update notifications Hides drawer-update entries from the bell ("Updated 5m ago"). Failed-save errors still surface so you don't miss something that didn't persist.
Notification sounds Plays a brief two-tone chime when a new notification arrives. Synthesized locally — no audio file. Debounced to avoid back-to-back tones.
Send test notification Pushes a fake Save failed entry into the bell — the same red notification a real save/restore/delete error would show. Useful for previewing the error chime and confirming the notification surface works.
Mark all memories as seen Clears the blue recently-updated marker from every memory across all wings in one click. Useful after a batch edit / mass sync / migration that bumped many cards at once. Storage is unaffected — this only updates your local seen map.

Keyboard shortcuts

Click a global binding to record a new key. Esc cancels recording. The editing-mode shortcuts below are universal conventions and are not remappable.

Global

Toggle maximize Enlarges the panel under focus — Detail when a memory is open, Memories otherwise.
Close overlay Dismisses sheets, menus, and the detail panel — one layer per press. Esc also still works as a built-in fallback.
Delete focused Removes the focused wing/room or the open memory. Always routes through the trash — reversible.
Edit memory Opens the inline editor for the currently-selected memory — same path as clicking the pencil button on the detail panel.

Tools

Writes, destructive ops, and advanced queries. Browsing has moved inline — rooms now show their tunnels as chips and the diary lives as its own wing.

Query an entity

See all facts touching an entity. Optionally pin a date or filter direction. Transitional — KG facts will surface inline on drawer details in 0.5.1.

Fact timeline

Chronological story of an entity, or every fact ever filed if you leave it blank. Transitional — will become a "See timeline" button on entity drawers in 0.5.1.

Tunnels

Browsing tunnels is now inline — every room nav row shows a chip with its connected rooms. Use this pane to create a new tunnel by hand or run an advanced multi-hop graph traversal.

Create tunnel

Most of the time you don't need this pane — click "+ Connect this room…" on any room nav row and the source is prefilled for you.

Traverse graph (advanced)

Walk the tunnel graph N hops from a starting room. Useful for discovering indirect connections that don't fit in the inline chip view.
